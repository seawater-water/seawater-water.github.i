首先指针是干什么的？

  • 指针变量是一个特殊的变量，他里面存放的值为一个地址为指针，通过“ * ”解引用为这个地址的值；
  • 指针变量类型有很多：
  • int *ptr; : 指针的类型是 int*
  • char *ptr; : 指针的类型是 char*
  • int *ptr;* : 指针的类型是 *int*
  • int (*ptr)[3]; : 指针的类型是 int(*)[3]
  • int *(*ptr)[4]; : 指针的类型是 int*(*)[4]
  • 指针变量所指向的类型也有很多，但与指针变量类型不一样：
  • int*ptr; : 指针所指向的类型是 int
  • char*ptr; : 指针所指向的的类型是 char
  • *int*ptr; : 指针所指向的的类型是 int*
  • int(*ptr)[3]; : 指针所指向的的类型是 int()[3]
  • int*(*ptr)[4]; : 指针所指向的的类型是 int*()[4]
  • 指针变量本身占据了 4 个字节的长度。
  • 关于“指针”和“指针变量”比较严格的说法是这样的：
    系统为每一个内存单元分配一个地址值，C/C++把这个地址值称为“指针”。如有int i=5;，存放变量i的内存单元的编号(地址)&i被称为指针。
    “指针变量”则是存放前述“地址值”的变量，也可以表述为，“指针变量”是存放变量所占内存空间“首地址”的变量（例如数组）(因为一个变量通常要占用连续的多个字节空间)。比如在int i=5;后有一句int p=&i;，就把i的指针&i赋给了*int *型指针变量p**，也就是说p中存入着&i。所以说指针变量是存放指针的变量。
    如对int *p=&i;的解释是：应该是声明一个**int *型指针变量p    ，将**&i这个地址(指针    )赋值给变量p(等价于int p; p = &i*)
  • “.”一般读作“的”，其前面是结构体变量，不是结构体类型
    ->读作“指向结构体的”，其前面一定是一个指向结构体的指针

指针可以比较大小吗？

  • 若两个指针指向同一数组的元素，则可以进行比较。指向前面的元素的指针变量“小于”指向后面的元素的指针变量。附加一下，若两个指针指向同一数组的元素，，则两个指针变量的值之差是两个指针之间的元素。

2022/10/20

  • 数组当中有很多的元素，结构中有很多的成员
  • 数组中的元素(a[0])类型必须一样，可以做左值，也可以做右值
  • 结构中的成员类型可以不一样

指针数组：

  • int *(arr[4]) = int*arr[4]（因为[]优先级比()高）

  • 解释：这是一个数组，数组中的每个元素是指针，本质是数组

  • 数组名是数组首元素的地址

  • ```c++
    void test20(){
    int num1 = 100;
    int num2 = 200;
    int num3 = 300;
    int num4 = 400;
    int *arr[5] = {&num1 ,&num2 , &num3 , &num4};
    for (int j = 0; j < 4 ; ++j) {
    cout<<*arr[j]<<endl;//100,200,300,400
    }
    cout<<arr[0]<<endl;//num1的地址;0x14eb1ffc88
    cout<<*arr[0]<<endl;//num1的值;100
    cout<<arr<<endl;//数组首元素的地址即num1的地址的地址;0x14eb1ffc50
    cout<<*arr<<endl;//num1的地址;0x14eb1ffc88
    }

数组首地址&首元素地址：

  • 数组首元素地址: &arr[0] == arr （数组名作为数组首元素地址）
  • arr+1跳过一个元素
  • 数组首地址: &arr
  • &arr+1跳过整个数组

数组指针：

  • int arr[5]={1,2,3,4,5};

  • int (*p)[5] = &arr;

  • p+1=&arr+1

15:57

  • &(取址符)只能对变量取地址，不能对常量取地址（数组名是符号常量，代表数组首元素的地址）
  • 例如&(arr+1）是错误的，但有个例外就是&arr他代表数组的首地址，这里的arr并不看为首元素地址，而是看为一个变量，变量类型为数组
